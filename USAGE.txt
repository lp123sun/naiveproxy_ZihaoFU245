Usage: naive --listen=... --proxy=...
       naive [/path/to/config.json]

Description:

  naive is a proxy that transports traffic in Chromium's pattern.
  It works as both a proxy client and a proxy server or together.

  Options in the form of `naive --listen=... --proxy=...` can also be
  specified using a JSON file:

    {
      "listen": "...",
      "proxy": "..."
    }

  Specifying a flag multiple times on the command line is equivalent to
  having an array of multiple strings in the JSON file.

  Uses "config.json" by default if run without arguments.

Options:

  -h, --help

    Shows help message.

  --version

    Prints version.

  --listen=LISTEN-URI

    LISTEN-URI = <LISTEN-PROTO>"://"[<USER>":"<PASS>"@"][<ADDR>][":"<PORT>]
    LISTEN-PROTO = "socks" | "http" | "redir"

    Listens at addr:port with protocol <LISTEN-PROTO>.
    Can be specified multiple times to listen on multiple ports.
    Default proto, addr, port: socks, 0.0.0.0, 1080.

    Note: redir requires specific iptables rules and uses no authentication.

      (Redirecting locally originated traffic)
      iptables -t nat -A OUTPUT -d $proxy_server_ip -j RETURN
      iptables -t nat -A OUTPUT -p tcp -j REDIRECT --to-ports 1080

      (Redirecting forwarded traffic on a router)
      iptables -t nat -A PREROUTING -p tcp -j REDIRECT --to-ports 1080

      Also activates a DNS resolver on the same UDP port. Similar iptables
      rules can redirect DNS queries to this resolver. The resolver returns
      artificial addresses that are translated back to the original domain
      names in proxy requests and then resolved remotely.

      The artificial results are not saved for privacy, so restarting the
      resolver may cause downstream to cache stale results.

  --proxy=PROXY

    PROXY = PROXY-CHAIN | SOCKS-PROXY
    PROXY-CHAIN = <PROXY-URI>[","<PROXY-CHAIN>] 
    PROXY-URI = <PROXY-PROTO>"://"[<USER>":"<PASS>"@"]<HOSTNAME>[":"<PORT>]
    PROXY-PROTO = "http" | "https" | "quic"
    SOCKS-PROXY = "socks://"<HOSTNAME>[":"<PORT>]

    Routes traffic via the proxy chain.
    The default proxy is a direct connection without proxying.
    The last PROXY-URI is negotiated automatically for Naive padding.

    If multiple proxies are specified, they must match the number of specified
    LISTEN-URIs, and each LISTEN-URI is routed to the PROXY matched by position.

    Limitations:
      * QUIC proxies cannot follow TCP-based proxies in a proxy chain.
      * The user needs to ensure there is no loop in the proxy chain.
      * SOCKS proxies do not support chaining, authentication, or Naive padding.

  --insecure-concurrency=<N>

    Use N concurrent tunnel connections to be more robust under bad network
    conditions. More connections make the tunneling easier to detect and less
    secure. This project strives for the strongest security against traffic
    analysis. Using it in an insecure way defeats its purpose.

    If you must use this, try N=2 first to see if it solves your issues.
    Strongly recommend against using more than 4 connections here.

  --tunnel-timeout=<SECONDS>

    After living for this timeout, a tunnel connection is retired: new streams
    are tunneled in new connections, and the old tunneled streams are forcibly
    closed by the idle timeout if idle or by the tunnel timeout if not idle.
    And the retired connection is closed when all tunneled streams are closed.
    This setting helps with CGNAT networks where long-lived connections are
    stuck, but will break long-lived TCP protocols, e.g. SSH.
    Default: 1800; 600 (Android).

  --idle-timeout=<SECONDS>

    After being idle for this timeout, tunneled streams are forcibly closed for
    timely cleanup of retired connections. When a stream is idle for long
    enough, it is assumed that the stream is unused and the breakage is mild.
    Default: 600; 300 (Android).

  --extra-headers=...

    Appends extra headers in requests to the proxy server.
    Multiple headers are separated by CRLF.

  --masque-udp=<true|false>

    Enables SOCKS5 UDP ASSOCIATE over CONNECT-UDP when the upstream proxy uses
    HTTP/2 or HTTP/3. Default: false.

  --masque-udp-path-template=<PATH_TEMPLATE>

    Overrides the CONNECT-UDP request path template. The template must begin
    with "/" and contain both {target_host} and {target_port}.

  --connect-udp-timeout=<SECONDS>

    Removes a CONNECT-UDP target flow after this many seconds without an
    admitted client datagram or an upstream datagram. Each UDP association
    schedules cleanup for its earliest flow deadline. Default: 120.

  --socks-udp-association-timeout=<SECONDS>

    Immediately closes a SOCKS5 UDP association after this many seconds without
    valid control activity, accepted client UDP traffic, or upstream UDP
    traffic. All owned CONNECT-UDP streams are reset. Default: 300.

  --udp-max-target-flows=<N>

    Maximum target endpoints per SOCKS5 UDP association. Each target consumes
    one upstream HTTP stream. This limit is independent of Chromium's
    MaxConnectionsPerProxy policy because naive's upstream requests explicitly
    ignore socket-pool limits. CONNECT and CONNECT-UDP streams use Chromium's
    common proxy-tunnel scheduler without reserving capacity for either type.
    When this limit is reached, datagrams for new targets are discarded rather
    than displacing established flows. Default: 128.

  --udp-max-queued-datagrams-per-flow=<N>

    Maximum datagrams waiting for one CONNECT-UDP target flow. Default: 16.

  --udp-max-queued-bytes-per-flow=<N>

    Maximum payload bytes waiting for one CONNECT-UDP target flow.
    Default: 262144.

  --udp-max-queued-sends=<N>

    Maximum response datagrams waiting to be sent to the SOCKS5 client.
    Default: 64.

  --udp-max-queued-send-bytes=<N>

    Maximum response bytes waiting to be sent to the SOCKS5 client.
    Default: 1048576.

    When a queue reaches either of its limits, naive discards the oldest
    waiting datagrams to bound latency and retain newer QUIC recovery traffic.
    A value of zero disables any of the five UDP limits. Limits apply per
    SOCKS5 UDP association; flow queue limits apply separately per target.

  --host-resolver-rules="MAP proxy.example.com 1.2.3.4"

    Statically resolves a domain name to an IP address.

  --resolver-range=CIDR

    Uses this range in the builtin resolver. Default: 100.64.0.0/10.

  --log=[<path>]

    Saves log to the file at <path>. If path is empty, prints to
    console. No log is saved or printed by default for privacy.

  --log-net-log=<path>

    Saves NetLog. View at https://netlog-viewer.appspot.com/.

  --ssl-key-log-file=<path>

    Saves SSL keys for Wireshark inspection.

  --no-post-quantum

    Overrides the default and disables post-quantum key agreement.
